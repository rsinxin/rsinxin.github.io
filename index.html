flex布局

Posted on 2018-11-01

flex布局是什么

Flex是Flexible Box的缩写，意为”弹性布局”，用来为盒状模型提供最大的灵活性。

1.定义一个容器为 flex 布局

注意，设为Flex布局以后，子元素的float、clear和vertical-align属性将失效。

基本概念

  1. 容器：采用 flex 布局的元素，称为Flex容器，简称 容器
  2. 项目：容器的所有子元素自动成为容器成员，称为 Flex 项目，简称 项目

容器默认存在两根轴：水平的主轴（main axis）和垂直的交叉轴（cross axis）。主轴的开始位置（与边框的交叉点）叫做main start，结束位置叫做main end；交叉轴的开始位置叫做cross start，结束位置叫做cross end。

项目默认沿主轴排列。单个项目占据的主轴空间叫做main size，占据的交叉轴空间叫做cross size。

容器的属性

在容器上主要有以下六个属性

  • flex-direction
  • flex-wrap
  • flex-flow
  • justify-content
  • align-items
  • align-content

flex-direction

flex-direction 属性决定主轴的方向

  • row (默认) 主轴为水平方向，起先为左端
  • row-reverse 主轴为水平方向，起点为右端
  • column 主轴为垂直方向，起点在上沿
  • column-reverse 主轴为垂直方向，起点在下沿

flex-wrap

flex-wrap 属性绝对项目如何换行

  • nowrap (默认) 不换行
  • wrap 换行，第一行在上方
  • wrap-reverse 换行，第一行在下方

flex-flow

flex-flow 属性是 flex-direction 和 flex-wrap 的简写

justify-content

justify-content 属性是定义项目在主轴的对齐方式

  • flex-start (默认) 沿着主轴向左对齐
  • flex-end 沿着主轴向右对齐
  • center 居中
  • space-between 两端对齐，项目之间的间隔相等
  • space-around 每个项目两边的间隔相等，两端的间隔是项目之间间隔的一半

align-items

align-items 属性定义项目在交叉轴的对齐方式

  • flex-start 与交叉轴起点对齐
  • flex-end 与交叉轴终点对齐
  • center 与交叉轴中点对齐
  • strctch (默认) 如果项目中未设置高度或者设置为auto，则占满容器高度
  • baseline 项目的第一行文字的基线对齐

align-content

align-content 属性定义多跟轴线的对齐方式，如果只有一根轴线，则该属性不起作用

  • flex-start 与交叉轴起点对齐
  • flex-end 与交叉轴终点对齐
  • center 与交叉轴中线对齐
  • space-between 在交叉轴两端对齐，项目之间间隔相等
  • space-around 每个项目在交叉轴方向两端距离相等，两端的间隔是项目之间间隔的一半
  • stretch (默认) 在交叉轴方向占据整个空间，如果除去所有行，还有剩余空间，则每一行平分剩余的空间

项目的属性

  • order
  • flex-gorw
  • flew-shrink
  • flex-basis
  • flex
  • align-self

order

order 定义项目的排序方式，默认为0，值越小，排列越靠前

flex-grow

flex-grow 定义项目的放大比例，有剩余空间的情况下，按比例等分剩余空间

默认为 0，不放大

flex-shrink

flex-shrink 定义项目的缩小比例，在空间不足时，项目将按比例缩小

默认为 1 ；当值为 1 时，此项目不缩小 ；负值无效

flex-basis

flex-basis 定义项目占据的主轴空间，浏览器会根据此属性，计算主轴是否还有多余空间

默认值 auto ，即项目的本来大小

flex

flex 属性是一个简写

默认值 为 none（0 0 auto）
快捷键 auto （1 1 auto）

align-self

align-self 属性代表允许项目与其他项目有不一样的对齐方式

  • flex-start 沿主轴开端对齐
  • flex-end 沿主轴终点对齐
  • center 沿主轴中点对齐
  • baseline 沿每个项目第一行文字的基线对齐
  • stretch 沿交叉轴方法拉伸到最大长度
  • auto 默认值，代表继承容器中的 align-items 属性，如果容器未设置该属性，则等同与设置stretch属性

构造函数中的 this 与 return

构造函数new一个新对象时，this发生了什么

1.创建了一个新对象 var obj = {}
2.将 Car 中的 this 关键字指向 obj
3.将 obj 的 proto 对象指向 Car 的 prototype 对象,这样 obj 就拥有了 Car原型中的方法
4.分别给 obj 的 color 和 owner 属性赋值
5.返回 obj 对象

构造函数的返回值 return

构造函数默认是返回this指向的对象，那么存在return时，有两种情况

  1. 返回值为 number，string，boolean，undefined，null 时，会忽视返回值，仍然返回this
  1. 返回值为一个 object 对象时，会返回这个对象

js 正则表达式

Posted on 2018-10-16

js 中正则表达式基本用法

正则表达式是由一个字符序列形成的搜索模式。
当你在文本中搜索数据时，你可以用搜索模式来描述你要查询的内容。正则表达式可以是一个简单的字符，或一个更复杂的模式，可用于所有文本搜索和文本替换的操作。

正则表达式的语法很简单，就是由主体和修饰符组成

/pattern(正则表达式主体)/modifiers(修饰符,可选)

修饰符

修饰符比较简单，先介绍修饰符比较好

修饰符 全称 描述
i ignore 对大小写不敏感
g global 全局查找
m multiple 执行多行匹配

方括号

方括号用于查找在某个范围内的字符

[abc] ：表示在方括号之间的任何字符
[^abc] : 表示不在方括号之间的任何字符
[0-9] : 表示 0~9 之间的任何字符
[a-z] : 表示 a~z 之间的任何字符
[A-Z] : 表示 A~Z 之间的任何字符
[A-z] : 表示 A~z 之间的任何字符

元字符

元字符是拥有特殊含义的字符

\w 查找单词字符
\W 查找非单词字符
\d 查找数字字符
\D 查找非数字字符
\s 查找空白字符
\S 查找非空白字符
\b 匹配单词边界
\B 匹配非单词边界

量词

n+ : 匹配 n 出现的次数为 >=1
n* : 匹配 n 出现的次数为 >=1
n? : 匹配 n 出现的次数为 0 或者 1
n{X} : 匹配 n 出现的次数为 =X
n{X,} : 匹配 n 出现的次数为 >=X
n{X,Y} : 匹配 n 出现的次数为 X<=n<=Y
n$ : 以 n 结尾的字符串
^n : 以 n 开头的字符串
?=n : 匹配任何其后紧接指定字符串 n 的字符串
?!n : 匹配任何其后没有紧接指定字符串 n 的字符串

正则表达式对象方法

exec : 检索字符串中指定的值,返回找到的值,并确定其位置
test : 检索字符串中指定的值,返回 true 或 false
/abc/.test(‘ ab abc acd’) 返回值为 true
toString : 返回正则表达式的字符串

支持正则表达式的String对象的方法

search : 检索与正则表达式相匹配的值
‘abcd’.search(‘bc’) 返回值为1
match : 找到一个或多个正则表达式的匹配
‘a1jka2’.match(/a[0-9]/g) 返回值为 [‘a1’,’a2’]
replace : 替换与正则表达式向匹配的字符串
‘ a b c ‘.replace(/^\s|\s$/g) 返回值为 ‘a b c’
split : 把字符串分割成字符串数组
‘ a b c ‘.split(/\s/g) 返回值为 [‘a’,’b’,’c’]

vue组件传值

vue传值

在使用Vue开发的项目过程中，经常提取一些组件来实现不同的功能。伴随着组件的使用和增多，使用一些方法在组件中传值和管理状态就显得特别重要，接下来介绍几种常见的组件传值的方法。

父子组件之间传值

父传子

  • 定义父组件，父组件通过 title 属性向子组件传值

  • 定义子组件，子组件通过 props 方法获取父组件传递的值。

子传父

  • 子组件通过 emit 事件向父组件传值

  • 父组件通过子组件触发的事件，接收传递的数据

不同组件之间传值，通过定义新的vue实例 Bus.js

  • 定义一个新的 vue 实例（Bus）专门用于传递数据（接收，传递数据）

  • 在组件一中定义方法名和传递数据，通过点击事件或者钩子函数触发 Bus实例的emit事件

  • 在组件二中接收组件一传递的数据

通过 SessionStorage 进行数据传递

  • 在组件1中设置 SessionStorage 的缓存数据

  • 在其他组件中就可以获取comp1 设置的cache1

通过路由带参数进行传值

  • 组件一通过在路由中设置传递参数

  • 在组件二中获取组件一传递的参数

使用vuex进行传值（在项目较复杂时使用）

Vuex是专为Vue.js应用程序开发的状态管理模式。主要集中管理应用中所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

浏览器端中的三种数据存储

Posted on 2018-10-15

浏览器端中的三种数据存储

cookie

浏览器最基本的本地储存方式就是cookie，cookie是存放再本地的一段文本，以键值对的形式保存，可以设计过期时间。cookie不适合大数据量储存，因为每请求一次页面，cookie会伴随着请求被发送给服务端，太大会影响数据传递效率。所以cookie大小限制在4k。

cookie简单使用

cookie的有效期：cookie默认情况下的有效期是很短暂的，一旦用户关闭浏览器，cookie保存的数据就会丢失。如果想要延长cookie的有效期，可以通过设置HTTP头信息中的cache-control属性的max-age值，或者修改HTTP头信息中的expires属性的值来延长有效期。

localStorage

html5中数据持久化存储的方式，以 key/value 的形式储存数据，大小限制在5M左右，是永久化储存，也会遵循同源策略。浏览器隐私模式下，localStorage是不可读取的。

localStorage简单使用

sessionStorage

localStorage是什么：在HTML5中，新加入了一个localStorage特性，这个特性主要是用来作为本地存储来使用的，解决了cookie存储空间不足的问题(cookie中每条cookie的存储空间为4k)，localStorage中一般浏览器支持的是5M大小，这个在不同的浏览器中localStorage会有所不同。
